### Lightweight alternative: DCO

If a full CLA is too heavy for a given contributor, the Project may instead accept
the [Developer Certificate of Origin](https://developercertificate.org/) via a
`Signed-off-by:` line (`git commit -s`). Note: the DCO certifies provenance but
does **not** grant relicensing rights — so it does not preserve the open-core
option. Prefer the CLA above when commercial licensing is a goal.
