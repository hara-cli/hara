# hara v0.89.0 — driven from 8 chat platforms, open all the way down

*2026-06-25*

**hara** is an open-source coding agent that runs like an engineering org. This release makes it reachable from
your chat apps, and completes the open-sourcing of the whole stack.

## What's new

### 🗨️ Chat gateway — 8 platforms
Run your local hara from a chat app with `hara gateway --platform <name>`:

**Telegram · WeChat · Discord · Feishu/Lark · Slack · Mattermost · Matrix · DingTalk**

- **Two-way images** — send the bot a photo and it *sees* it (inline for a vision model, or described via your
  configured vision model); ask it to send a file/image back and it does — in plain conversation, no command.
- **Connects out** (long-poll / WebSocket) — no public webhook URL, runs fine on your laptop behind NAT.
- Per-chat resumable sessions, project roaming (`/cd`), voice in/out (where supported).
- Adding a platform is one adapter file (the `ChatAdapter` seam) — the gateway core is platform-agnostic.

See **[docs/gateway.md](../gateway.md)** for per-platform setup.

### 🔓 Open all the way down
- The hara **CLI** is Apache-2.0 / MIT.
- The **control plane** ([hara-control](https://github.com/hara-cli/hara-control)) — device enrollment, token
  issue/revoke, fleet view, governance — is now **open source too (Apache-2.0)**. Even the token/key layer is
  open: your upstream provider keys never leave your infrastructure. **No obfuscation** — every shipped line is
  readable. We make money by *operating* it (managed/hosted), not by withholding it.
